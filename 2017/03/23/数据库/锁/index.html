    数据库中常见的锁

    间隙锁

    模拟数据

    id name
    1 aa
    3 bb
    8 cc

    事务一:

        begin;
    select * from test_deadlock where id=5 for update;
    
    commit;
    ROLLBACK;

    事务二:

        begin;
    -- 插入（3, 5）之间的数据 锁等待
    insert into test_deadlock VALUES(4, 'aa')
    
    -- 插入（5, 8）之间的数据 锁等待
    insert into test_deadlock VALUES(7, 'aa')
    
    -- 插入（8, ++）之间的数据 正常
    insert into test_deadlock VALUES(9, 'aa')
    
    commit;
    ROLLBACK;

    现象分析

    锁不存在的数据时会产生间隙锁
    间隙锁范围: （3, 5） （5, 8）

    锁已存在的数据时，不会产生间隙锁（GAP）

    间隙锁产生原因

    为了解决幻读问题，早期的rr还是会产生幻读问题的，后来引入了GPA间隙锁才解决了幻读的问题

    死锁场景

    获取锁顺序不一致导致死锁

    两条update语句即可导致死锁

    id name age
    1 aa 10
    3 bb 20
    8 cc 30

    事务一:

        begin;
    -- 实际SQL
    update test_deadlock set age=age+10 where id=1 or id=3;
    
    -- 拆分SQl
    update test_deadlock set age=age+10 where id=1 
    
    update test_deadlock set age=age+10 where id=3
    
    commit;
    ROLLBACK;

    事务二:

        begin;
    -- 实际SQL
    update test_deadlock set age=age+10 where name='bb' or name='aa';
    
    -- 拆分SQl
    update test_deadlock set age=age+10 where name='bb';
    
    update test_deadlock set age=age+10 where name='aa';
    
    commit;
    ROLLBACK;

    由于这种模拟方法比较随机，不能保证一定复现死锁场景，我们暂且把它拆分成两条SQL
    拆分只会就很容易看出来如何产生死锁
    事务一: 获取锁id=1
    事务二：获取锁id=3
    事务一: 等待事务二释放锁id=3
    事务二: 等待事务一释放锁id=1
